# Dynamics Formulation

A technical reference on state-space modeling, Markov processes, and Monte Carlo simulation for stochastic decision systems.

---

## 1. The State Space ($S$)

The **State Space** ($S$) is the mathematical set of all possible situations or configurations a system can possibly be in. It is the universe of all valid "snapshots" of an environment.

> **Example — Pairs Trading (Cointegration):** The hedge ratio between two cointegrated assets, which is not directly observable and varies over time, is modeled as a state. The price spread between the assets is the observed noisy output.

```python
import numpy as np

PRICE_STATES  = ["low", "high"]
GEN_STATES    = ["low", "high"]
REGIME_STATES = ["normal", "peak", "volatile"]

PRICE_IDX  = {s: i for i, s in enumerate(PRICE_STATES)}
GEN_IDX    = {s: i for i, s in enumerate(GEN_STATES)}
REGIME_IDX = {s: i for i, s in enumerate(REGIME_STATES)}
```

## 2. Markov Property

A system has the **Markov Property** if the future depends only on the present state, and completely ignores the past.

Mathematically, the probability of transitioning to the next state $S_{t+1}$ depends only on the current state $S_t$, not on the sequence of states that preceded it:

$$P(S_{t+1} \mid S_t, S_{t-1}, S_{t-2}, \ldots) = P(S_{t+1} \mid S_t)$$

If our state space is defined well enough, the current state encapsulates everything one needs to know to predict the future.

---

## 3. Markov Decision Process (MDP)

An **MDP** is a mathematical framework for modeling decision-making in situations where outcomes are partly random and partly under the control of a decision-maker (the *agent*).

An MDP is formally defined by a 5-tuple:

$$(S, A, P, R, \gamma)$$

- **$S$ (State Space):** The set of all possible states, as discussed above.
- **$A$ (Action Space):** The set of all possible actions the agent can take. For a robotic vacuum, $A$ might be $\{\text{Move Forward}, \text{Turn Left}, \text{Turn Right}, \text{Dock}\}$.
- **$P$ (Transition Probability):** $P(s' \mid s, a)$ is the probability of ending up in state $s'$ given that the agent took action $a$ in state $s$. This is where the physics of the real world — friction, noise, sensor errors — comes into play. If the vacuum tries to move forward, it might slip and end up slightly to the left.
- **$R$ (Reward Function):** $R(s, a)$ is the immediate numeric reward received after transitioning from state $s$ via action $a$. This is how we define the goal. Sucking up dirt is $+10$. Falling down the stairs is $-100$.
- **$\gamma$ (Discount Factor):** A number between 0 and 1. It determines the present value of future rewards. A $\gamma$ close to 0 makes the agent incredibly short-sighted (only caring about immediate rewards), while a $\gamma$ close to 1 makes the agent strive for long-term optimization.

### Why Discount?

1. **Mathematical convergence:** If an MDP runs infinitely, summing up rewards would result in infinity. Discounting forces the infinite sum to converge to a finite number.
2. **Uncertainty:** Because the future is stochastic, a guaranteed reward now is better than a probable reward later.

### The Objective

The ultimate goal of solving an MDP is to find an **Optimal Policy** ($\pi^*$). A policy is simply a mapping from states to actions:

$$\pi: S \rightarrow A$$

It acts as an instruction manual: *"If you find yourself in state $s$, take action $a$."* The optimal policy is the one that maximizes the expected cumulative discounted reward over time.

## 4. Discrete Time Markov Chain (DTMC)

A **DTMC** is an MDP with the "decision" and "reward" parts stripped out.

It models a system that transitions from one state to another over discrete time steps purely based on probability, with no external agent choosing actions and no rewards being collected. It just evolves on its own.

It is defined by only two things:

- A set of **States** ($S$).
- A **Transition Probability Matrix** ($P$).

> **Example:** Imagine modeling the operating state of a server. It has three states: *Working*, *Degraded*, and *Failed*. Every hour (discrete time step), there is a probability it stays in its current state or moves to another. No one is taking "actions" to change it, and there are no "rewards." We just want to watch the probabilities evolve to answer questions like: *"If the server is working now, what is the probability it will be in the Failed state 5 hours from now?"*

```python
REGIME_TRANSITION = np.array([
    [0.70, 0.20, 0.10],   # normal   -> normal 0.70, peak 0.20, volatile 0.10
    [0.30, 0.50, 0.20],   # peak     -> normal 0.30, peak 0.50, volatile 0.20
    [0.40, 0.20, 0.40],   # volatile -> normal 0.40, peak 0.20, volatile 0.40
])
```

## 5. Regime Switching Model

A **Regime Switching Model** assumes that the data is generated by an underlying system that jumps between different unseen *regimes* or *states*.

> **The Connection:** The underlying regime jumps from state to state following a Discrete Time Markov Chain (DTMC).
>
> **The Difference:** You cannot directly see the Markov Chain's state. You can only observe the noisy data it spits out.

> **Example:** The stock market jumps between a *Bull Market* regime and a *Bear Market* regime.
>
> - You cannot directly observe the "true" state of the economy (it is hidden).
> - You can observe daily stock returns.
> - The underlying market state transitions between Bull and Bear according to a Markov Chain.
>
> When it is in the *Bull* state, it generates data with high mean and low volatility. When in the *Bear* state, it generates data with negative mean and high volatility.

```python
PRICE_TRANSITION = {
    "normal":   np.array([
        [0.80, 0.20],   # low  -> low 0.80, high 0.20  (stable, low-price regime)
        [0.30, 0.70],   # high -> low 0.30, high 0.70
    ]),
    "peak":     np.array([
        [0.10, 0.90],   # low  -> high with high probability
        [0.05, 0.95],   # high -> stays high almost certainly
    ]),
    "volatile": np.array([
        [0.50, 0.50],   # uniform — price can go either way
        [0.50, 0.50],
    ]),
}

GEN_TRANSITION = {
    "normal":   np.array([
        [0.60, 0.40],   # low  -> stays low 0.60, goes high 0.40
        [0.50, 0.50],   # high -> equal chance
    ]),
    "peak":     np.array([
        [0.80, 0.20],   # low generation during peak periods (cloud cover / high demand)
        [0.70, 0.30],   # high -> likely drops
    ]),
    "volatile": np.array([
        [0.50, 0.50],   # uniform
        [0.50, 0.50],
    ]),
}
```

## 6. Adding Black Swan Events to the MDP

I added `SPIKE_PROB = 0.07` to model **rare but high-impact market events** that the normal transition matrices cannot capture.

In the current setup, `PRICE_TRANSITION` handles typical behavior (smooth movement between low/high prices under each regime). But real markets — especially in energy — have sudden spikes due to shocks (demand surge, outages, policy changes). A pure Markov matrix would make these spikes either too frequent or impossible.

So I manually introduced a **Bernoulli trial**:

- With probability $0.07$ → force a jump to high price
- With probability $0.93$ → follow normal transition dynamics

This does two important things:

1. **Adds realism** — captures fat-tail / jump behavior seen in real markets.
2. **Improves policy robustness** — the agent learns to prepare for sudden profitable (or risky) spikes instead of overfitting to smooth transitions.

## 7. Monte Carlo Execution (Samplers)

This section turns the static transition matrices into a **stochastic simulation engine**. Instead of deterministically moving between states, the next state is sampled using probability distributions.

The execution order:

1. **Regime** is sampled first using `rng.choice` over `REGIME_TRANSITION`.
2. **Price** is sampled conditionally on the next regime, including a Bernoulli spike override (`rng.random() < SPIKE_PROB`) to model rare events.
3. **Generation** is sampled conditionally on the next regime.

All components are combined in a single function that respects the correct order of dependencies (**regime → price/gen**), ensuring the system follows the designed dynamics.

```python
def sample_next_regime(current_regime: str, rng: np.random.Generator) -> str:
    idx = REGIME_IDX[current_regime]
    next_idx = rng.choice(len(REGIME_STATES), p=REGIME_TRANSITION[idx])
    return REGIME_STATES[next_idx]

def sample_next_price(current_price: str, next_regime: str, rng: np.random.Generator) -> str:
    # Rare spike: volatile regime only — overrides normal transition
    if next_regime == "volatile" and rng.random() < SPIKE_PROB:
        return "high"
    idx = PRICE_IDX[current_price]
    next_idx = rng.choice(len(PRICE_STATES), p=PRICE_TRANSITION[next_regime][idx])
    return PRICE_STATES[next_idx]

def sample_next_exogenous(price: str, gen: str, regime: str, rng: np.random.Generator):
    next_regime = sample_next_regime(regime, rng)
    next_price  = sample_next_price(price, next_regime, rng)
    next_gen    = sample_next_gen(gen, next_regime, rng)
    return next_price, next_gen, next_regime
```
